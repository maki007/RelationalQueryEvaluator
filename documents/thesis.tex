Abstract:
This thesis deals with the design and implementation of optimizer and transformer of relational queries.
Firstly, the thesis describes database compiler theory.
Following chapter presents data structures and algorithm used in implemented tool. Final chapter specifies important implementation details.
Part of the thesis was the selection of used relational algebra operators and design of appropriate input.
Input of implemented software is query written in XML file in form of relational algebra.
Query is optimized  and transformed into physical plan which will be executed in parallel framework Bobox.
Developed compiler outputs physical plan written in Bobolang language, which serves as input for Bobox.


Keywords:
SQL, Compiler, Relational algebra, optimizer, Bobox

\section*{Attachment on CD}
The genereted documentation can be found in folder \texttt{Relational\-Query\-Eva\-lua\-tor/\-Doxygen/\-html/\-html}. Folder \texttt{Thesis} contains text of this thesis.

Software was developed in Visual studio and its solution can be found in folder \texttt{Relational\-Query\-Evaluator}. Recommend tool for compiling source codes is Visual Studio 2013. We used external library $Xerces C++$ version 3.1.1, which is located in folder \texttt{Relational\-Query\-Evaluator/\-externals}. 

Test queries can be found in folder \texttt{Relational\-Query\-Evaluator/\-Relational\-Query\-Evaluator/\-data}. Files with suffix \text{.xml} contains test queries. Compiler generates the following outputs for every processed file (for example named \texttt{query.xml}) containing query:   
\begin{enumerate}
\item \texttt{query.xml.\_1.txt} -- Dot representation of the input algebra tree.
\item \texttt{query.xml.\_2.txt} --  Dot representation of the input algebra tree after semantic analysis and grouping phase.
\item \texttt{query.xml.\_3.txt} -- Dot representation of the optimized algebra tree.
\item \texttt{query.xml.\_4.txt} -- Dot representation of the best physical plans with unresolved sort parameters.
\item \texttt{query.xml.\_5.txt} -- Dot representation of the best final physical plans.
\item \texttt{query.xml.\_6.bbx} -- Bobolang representation of the best physical plan.
\end{enumerate}
In folder with test queries we can find generated PNG images from outputs of queries. Description of image output can be found in Chapter~\ref{implementation}.
Provided Windows binaries an located in folder \texttt{Binaries}.

Program usage: 
\begin{itemize}
\item \texttt{RelationalQueryEvaluator.exe inputfile}
\end{itemize}
 Every line of the input file should contain name of the file with relational query.
